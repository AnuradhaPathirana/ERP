# UI/UX Engineering and Optimization Guidelines

This document establishes the strict UI/UX standards for the SaaS ERP application. Every component, view, layout, and workflow generated by the AI or built by developers must comply with these guidelines to ensure the system is easy to use, highly optimized, and incredibly fast.

---

## 1. Usability and Ease of Use

Enterprise users interact with ERP systems for hours daily. The focus must be on efficiency, cognitive clarity, and minimizing user fatigue.

### Navigation and Layout Predictability

- **Persistent Layout:** The left sidebar navigation and top navbar must remain fixed. Users should always know their exact location in the system.
- **Standard Layout Patterns:** Place high-frequency actions (e.g., "Create Invoice", "Add Product") in consistent top-right button positions on all index screens.
- **Breadcrumbs:** Include breadcrumbs on all sub-pages (e.g., Inventory > Products > Edit Product) for seamless back-navigation without losing application state.

### Data Entry Efficiency

- **Keyboard-First Design:** All forms must support standard tabbing navigation. High-velocity data-entry screens (like Point of Sale or Stock Adjustments) must handle the Enter key to save or move to the next logical field.
- **Smart Defaults:** Pre-fill form fields wherever possible (e.g., current date, default warehouse, primary tax rate) to eliminate unnecessary clicks.
- **Inline Validation:** Validate forms dynamically on field blur rather than waiting for a full form submission. Show clear, contextual error messages beneath the input.

---

## 2. Performance and Fast Loading Times

An ERP must feel instantaneous. Page changes and data loads should execute in under 300 milliseconds to maintain user productivity.

### Technical Performance Matrix

| Area              | Target Metric          | Strategy to Achieve                                                                                                 |
| :---------------- | :--------------------- | :------------------------------------------------------------------------------------------------------------------ |
| Initial Page Load | Under 1.5 seconds      | Route-based code splitting and lazy loading of React components.                                                    |
| API Data Fetching | Under 200 milliseconds | Implement client-side caching (e.g., TanStack Query or SWR) for static tables like Units and Categories.            |
| Large Data Tables | Infinite or Paginated  | Server-side pagination with a strict limit of 25-50 rows per page. Never pull entire datasets into the React state. |

### Asset and State Optimization

- **Iconography:** Use exclusively raw SVG components via the `lucide-react` library. Do not load monolithic icon fonts or heavy asset packs.
- **Optimized Bundling:** Bundle features strictly by module. Ensure loading the Inventory module does not download the code or assets for the Finance or HR modules.
- **Optimized Payloads:** Clean backend API resources to return only fields required by the UI. Avoid sending deeply nested database relational data unless explicitly required.

---

## 3. Screen Optimization and Data Density

ERPs require high data density. The interface must maximize screen real estate without cluttering the user experience.

### Layout Fluidity and Breakpoints

- **Desktop Maximization:** Optimize layouts for standard 1920x1080 and 1366x768 business monitors. Use a wide canvas with controlled container caps (`max-w-7xl` or dynamic viewports).
- **Responsive Downscaling:** While complex data entry is built for desktops, dashboards and approval lists must degrade gracefully down to mobile screens using standard Tailwind breakpoints (`sm`, `md`, `lg`, `xl`).

### Standard Enterprise Data Tables

- **Compact Padding:** Use tight vertical padding (`py-2` or `py-3` in Tailwind) on data tables to display more rows above the fold.
- **Sticky Elements:** Table headers must remain sticky at the top when scrolling vertically. Action columns (Edit/Delete) must remain sticky on the right during horizontal scrolling on smaller displays.
- **Text Truncation:** Use text truncation (`truncate` or `line-clamp-1`) for long text fields like product descriptions, supplemented by hover tooltips to show the full content.

---

## 4. Visual Standards and Component Rules

- **Typography:** Use a clean, highly readable system font stack (e.g., Inter, Segoe UI, system-ui). Maintain strict text hierarchy (Headers at `text-xl` or `text-2xl` bold, Body at `text-sm`).
- **Color Usage:** Stick to a professional slate palette (`bg-slate-50`, `text-slate-900`). Use accent colors strictly for status feedback: Red for alerts/out-of-stock, Green for active/success, Amber for warnings/low-stock.
